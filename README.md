# Image Processing Examples

This repository contains a set of modular image-processing example projects (C++ / CMake) demonstrating common operations such as grayscale conversion, thresholding, histograms, spatial filtering, segmentation, morphology and DCT. Each example lives in its own folder with source, data and build output produced by CMake.

**Quick summary**
- **Languages:** C++
- **Build system:** CMake (projects can be generated for Visual Studio on Windows)
- **Common dependency:** OpenCV (projects link against OpenCV components)

**Prerequisites**
- **CMake** (>= 3.10 recommended)
- **A C++ toolchain** (Visual Studio on Windows is used in generated builds)
- **OpenCV** built/installed and visible to CMake (projects expect OpenCV and copy DLLs on Windows)

**How to build (Windows / Visual Studio)**
1. Create a build folder (or use the provided `build/` folder inside each example):

```powershell
mkdir build
cd build
cmake ..
# or generate a specific generator, e.g.:
cmake -G "Visual Studio 17 2022" ..
``` 

2. Open the generated solution (`*.sln`) in Visual Studio and build, or build from command line with `cmake --build . --config Release`.

3. Run the produced executable. Some examples include `run.bat` or custom post-build copy steps for OpenCV DLLs.

**Notes about build scripts and linking**
- The per-example `src/CMakeLists.txt` files reference OpenCV targets and components. Ensure OpenCV is installed and CMake can find it (e.g., set `OpenCV_DIR` or install OpenCV system-wide).
- On Windows, many CMake files include logic to copy OpenCV DLLs to the executable output folder after build so the binary can run without adjusting PATH.

**Running examples**
- Typical flow inside an example folder:

```powershell
cd 01_grayscale
mkdir build
cd build
cmake ..
cmake --build . --config Release
# then run the executable from the generated output folder (or use run.bat if present)
```

**Extending / Adding examples**
- Add a new folder `0X_name/` with `src/`, optional `data/`, and a `CMakeLists.txt` that creates an executable and links to OpenCV targets.

**Where to look in the codebase**
- Example program entry points are in `*/src/main.cpp` files.
- Helper utilities are placed alongside examples in `*/src` (e.g., `Timer.h`, `imshow_multiple.h`).
- Build artifacts are placed in the `build/` subfolders (generated by CMake).
